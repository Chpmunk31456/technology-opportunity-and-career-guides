# Accessibility Statement and Review Workflow

## Commitment

Cybersecurity Career and Practical Guides is intended to be usable by people with different abilities, technologies, learning preferences, languages, and levels of professional experience.

Accessibility improvement is ongoing. Publication does not represent a claim that every file formally conforms to WCAG 2.2, PDF/UA, or another accessibility standard.

## Formats

- Markdown supports lightweight browser reading.
- PDF supports convenient reading and distribution.
- DOCX supports editing, notes, training, and assistive-technology workflows.

When one format creates a barrier, try another available format and report the problem.

## Review Checklist

Each guide should be checked for:

1. Correct document language and descriptive title
2. Logical heading hierarchy and reading order
3. Meaningful link text
4. Alternative text for meaningful images
5. Decorative images marked appropriately
6. Sufficient color contrast
7. Tables with identified headers
8. Lists created as real lists
9. Keyboard-accessible navigation
10. Usability at 200 percent zoom
11. Clear language and explained abbreviations
12. Tagged PDF structure and bookmarks when appropriate

## Review Status

- **Not yet evaluated**
- **Structural and automated checks completed**
- **Manual accessibility review completed**

Automated checks do not prove full accessibility.

## Correction Workflow

1. Record the guide, language, version, format, page, and barrier.
2. Describe the expected accessible behavior.
3. Correct the source document first when possible.
4. Regenerate related formats.
5. Recheck the affected requirement.
6. Record the correction in the changelog.

Disability or medical disclosure is never required. Do not include sensitive personal information in a public report.

Issues that block essential content, navigation, warnings, or downloads receive priority over cosmetic improvements.

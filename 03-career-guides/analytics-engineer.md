## What this role does

Transforms raw warehouse data into trusted, documented datasets for analysis and decisions.

## Core skills to develop

SQL, data modeling, testing, version control, business context.

Human skills also matter: clear writing, listening, teamwork, ethical judgment, curiosity, documentation, and the ability to explain technical choices to nontechnical people.

## A realistic entry path

1. Learn the basic vocabulary and purpose of the field.
2. Complete one free introductory course.
3. Practice with a small, legal, and clearly documented project.
4. Publish the project on GitHub without secrets, personal data, or unauthorized material.
5. Ask for feedback from a mentor or professional community.
6. Search official employer, internship, apprenticeship, fellowship, and open-source program pages.
7. Tailor a truthful resume to the actual requirements.

A degree can help in some markets, but it is not the only learning pathway. Technical school, apprenticeships, certifications, work experience, community college, independent study, and public portfolios may also provide evidence. Requirements differ by employer and country.

Use official vendor websites, recognized package managers, or the project's official source repository. Avoid unofficial download sites. Review licensing, system requirements, privacy terms, and cloud billing before installation or account creation. For cloud free tiers, set budgets and remove resources when the exercise is complete.

## First portfolio project

Create a small analytics project with tested transformations and a data dictionary.

Your README should explain the problem, scope, tools, setup, results, tests, limitations, accessibility, security or privacy considerations, and what you would improve.

## 30-day plan

- Week 1: Learn fundamentals and create a vocabulary list.
- Week 2: Follow one guided lab and reproduce it independently.
- Week 3: Begin the portfolio project and keep a learning journal.
- Week 4: Test, document, publish, and request feedback.

## 60-day plan

Complete a second project that adds teamwork, automation, security, data quality, accessibility, or cloud deployment. Review at least 20 real job descriptions and record repeated skills.

## 90-day plan

Complete a third project, improve your GitHub profile, prepare a targeted resume, practice interviews, and apply selectively to legitimate opportunities for which you meet the mandatory requirements.
